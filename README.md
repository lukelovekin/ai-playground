# ai-playground

A collection of agentic AI experiments and course projects exploring LLM workflows, tool use, multi-agent systems, and autonomous agents.

## Projects

### [`research_agent`](./research_agent)
A three-step reflective writing workflow: a draft is generated by one model, critiqued by a reasoning model (o4-mini), then revised by the original model incorporating the feedback. Demonstrates how chaining models with different strengths improves output quality compared to a single-pass generation.

**Pattern:** Reflection

**Tools:** **AISuite**, **OpenAI** (GPT-4o, o4-mini)

### [`reflection_pattern`](./reflection_pattern)
Implements the reflection pattern in two contexts:
- **Chart generation** — GPT-4o-mini generates matplotlib chart code, a vision model (o4-mini or Claude) inspects the rendered image and critiques it, then the code is revised and re-rendered.
- **SQL generation** — an LLM translates a natural language question into SQL, the query is executed against a SQLite database, and the result (or error) is fed back for the model to self-correct.

**Pattern:** Reflection, Self-correction

**Tools:** **AISuite**, **OpenAI** (GPT-4o-mini, o4-mini), **Anthropic** (Claude), **matplotlib**, **SQLite**

### [`tool_use`](./tool_use)
Explores how LLMs call Python functions as tools via AISuite. Covers automatic schema generation from type hints vs. manually defined JSON schemas, the difference in how `max_turns` is handled between the two, and a multi-tool email assistant that reads, drafts, and sends emails using tool calling.

**Pattern:** Tool calling / Function calling

**Tools:** **AISuite**, **OpenAI**

### [`research_agent_tools`](./research_agent_tools)
An agentic research pipeline that accepts a research question, runs parallel searches across arXiv (academic papers) and Tavily (web), synthesises a structured report, reflects on the output to identify gaps, revises it, then converts the final report to styled HTML for presentation.

**Pattern:** Agentic tool use, Reflection

**Tools:** **OpenAI**, **Tavily** (web search), **arXiv API**

### [`sql_agent`](./sql_agent)
A natural language SQL agent that connects to a MySQL database (Chinook music dataset) and answers free-form questions by generating and executing SQL queries. Built with LangChain's `create_sql_agent` using the ReAct framework — the agent reasons step-by-step, inspects the schema, writes a query, runs it, and returns the result. Accepts prompts via CLI argument.

**Pattern:** Agentic tool use, ReAct

**Tools:** **LangChain**, **IBM Watsonx** (Granite), **MySQL**

### [`component_evaluation`](./component_evaluation)
A component-level evaluation harness for an agentic research pipeline. Runs the retrieval step in isolation, checks whether the URLs returned come from a predefined list of trusted domains, and computes a PASS/FAIL accuracy metric — demonstrating how to evaluate individual agent components without running the full pipeline.

**Pattern:** Agent evaluation

**Tools:** **AISuite**, **Tavily** (web search)

### [`multi_agent_pipeline`](./multi_agent_pipeline)
A multi-agent research system where four specialised agents collaborate via an orchestrator: a **planner** breaks the topic into sub-questions, a **researcher** queries Tavily for each, a **writer** drafts a structured report from the findings, and an **editor** polishes the final output. Each agent is a separate LLM call with a distinct system prompt and role.

**Pattern:** Multi-agent orchestration

**Tools:** **AISuite**, **Tavily** (web search)

### [`autonomous_agents`](./autonomous_agents)
Two autonomous agent patterns:
- **Market research pipeline** — a team of agents produces a full product campaign: a researcher gathers market data via Tavily, DALL-E generates product imagery, a copywriter writes marketing copy using the image as multimodal input, and an executive agent compiles everything into a final report.
- **Customer service agent** — a code-as-action agent that handles natural language inventory queries by dynamically writing and executing DuckDB/TinyDB queries against a live sunglasses inventory, rather than calling pre-defined tools.

**Pattern:** Multi-agent, Code-as-action

**Tools:** **OpenAI** (GPT-4o, DALL-E), **AISuite**, **DuckDB**, **TinyDB**, **Tavily** (web search)

### [`translator-with-voice-and-watsonx`](./translator-with-voice-and-watsonx)
A voice-enabled Flask translation app powered by IBM Watsonx. User audio is transcribed via IBM Watson STT, the transcript is sent to a Watsonx LLM (Mistral) for translation or query response, and the result is synthesised back to speech via IBM Watson TTS — returned as base64-encoded audio alongside the response text.

**Pattern:** STT → LLM → TTS pipeline

**Tools:** **Flask**, **IBM Watsonx** (Mistral), **IBM Watson** (STT, TTS)

### [`chatapp-voice-openai`](./chatapp-voice-openai)
A voice-enabled Flask chat app where user audio is transcribed via IBM Watson STT, the transcript is processed by OpenAI GPT, and the response is synthesised back to speech via IBM Watson TTS. The `/speech-to-text` endpoint returns a JSON transcript; the `/process-message` endpoint returns both the response text and a base64-encoded WAV for playback in the browser.

**Pattern:** STT → LLM → TTS pipeline

**Tools:** **Flask**, **OpenAI** (GPT), **IBM Watson** (STT, TTS)

### [`genai_flask_app`](./genai_flask_app)
A Flask web app with a chat interface that lets users send a message and select which IBM Watsonx model to respond. Each model (Llama, Granite, Mistral) uses a model-specific prompt template and returns a structured JSON response containing a summary, a sentiment score (0–100), and a suggested reply. Static assets and templates were boilerplate generated by Claude.

**Pattern:** Prompt engineering, Structured output

**Tools:** **Flask**, **LangChain**, **IBM Watsonx** (Llama, Granite, Mistral)

### [`icebreaker`](./icebreaker)
A CLI chatbot that takes a LinkedIn profile URL, scrapes the profile via the ProxyCurl API, chunks and embeds the data into a vector index, then generates three interesting facts about the person as a conversation starter. Users can follow up with natural language questions answered by RAG over the indexed profile data.

**Pattern:** RAG

**Tools:** **LlamaIndex**, **IBM Watsonx** (WatsonxLLM, WatsonxEmbeddings), **ProxyCurl API**

### [`cal_coach_app`](./cal_coach_app)
A Flask web app that acts as an AI nutrition coach. Users upload a food image and optionally add a query; the image is base64-encoded and sent alongside a nutritionist system prompt to Llama 4 Maverick (IBM Watsonx). The model returns a structured breakdown covering food identification, portion sizes, calorie estimates, macro/micronutrient breakdown, and a health evaluation — formatted as HTML and rendered in the browser.

**Pattern:** Vision, Prompt engineering

**Tools:** **Flask**, **IBM Watsonx** (Llama 4 Maverick), **Pillow**

### [`vison_based_analysis`](./vison_based_analysis)
A Gradio fashion analysis app that takes an uploaded outfit image, encodes it into a vector, and finds the closest match in a pre-built Taylor Swift style embeddings dataset via cosine similarity. The matched item and related outfit pieces are passed alongside the user's image to a Llama vision model (IBM Watsonx) which generates a detailed breakdown of garments, fabrics, colours, and styling — with brand and pricing details where available.

**Pattern:** Vision, Vector similarity search

**Tools:** **Gradio**, **IBM Watsonx** (Llama Vision), **pandas**, **Pillow**

### [`faiss_yt_summariser`](./faiss_yt_summariser)
A Gradio web app that takes a YouTube URL, extracts the English transcript via the YouTube Transcript API, chunks it with LangChain's text splitter, and embeds the chunks into a FAISS vector index using IBM Watsonx SLATE embeddings. Users can generate a concise video summary or ask free-form questions answered by RAG — retrieving the most relevant transcript chunks and passing them to IBM Granite for generation.

**Pattern:** RAG, Summarisation

**Tools:** **Gradio**, **FAISS**, **LangChain**, **IBM Watsonx** (Granite, SLATE embeddings), **YouTube Transcript API**

### [`stt_and_analyzer`](./stt_and_analyzer)
Two scripts for audio transcription using OpenAI's Whisper model. `simple_speech2text.py` downloads an MP3 from a remote URL and transcribes it locally via the Whisper pipeline. `speech_analyzer.py` extends this into a Gradio web app — users upload an audio file, Whisper transcribes it, and an IBM Watsonx LLM (Llama) summarises the key points from the transcript.

**Pattern:** STT, Summarisation

**Tools:** **Transformers** (Whisper), **Gradio**, **IBM Watsonx** (Llama), **LangChain**

### [`image_caption`](./image_caption)
Automatic image captioning using Salesforce's BLIP model. Includes a standalone script and a Gradio web interface where users can upload an image and receive a generated caption in real time — no API key required, runs fully locally via HuggingFace Transformers.

**Pattern:** Vision, Image captioning

**Tools:** **Transformers** (Salesforce BLIP), **Gradio**, **Pillow**

### [`interactive_food_search`](./interactive_food_search)
A CLI food recommendation chatbot backed by ChromaDB vector similarity search. Food items are loaded from a JSON dataset, embedded, and stored in a ChromaDB collection. User queries are matched by semantic similarity and results are ranked with match scores, cuisine type, calorie info, and related search suggestions.

**Pattern:** RAG, Vector similarity search

**Tools:** **ChromaDB**, **IBM Watsonx**

### [`sentiment_analysis`](./sentiment_analysis)
A Flask web app that calls the IBM Watson NLP REST API to perform emotion detection on user-submitted text. Returns scores for anger, disgust, fear, joy, and sadness, and identifies the dominant emotion. Includes unit tests and error handling for blank or invalid input.

**Pattern:** NLP, Emotion detection

**Tools:** **Flask**, **IBM Watson NLP API**

### [`notebooks`](./notebooks)
Standalone reference notebooks:
- **`langchain_fundamentals`** — covers models, chat messages, prompt templates, output parsers, document loaders, text splitters, embeddings, vector stores, retrievers, memory, chains (LLMChain and LCEL), and agents with the ReAct framework.
- **`prompt_engineering_and_templates`** — basic, zero-shot, one-shot, few-shot, chain-of-thought, and self-consistency prompting, plus LangChain prompt templates applied to summarisation, QA, classification, code generation, and role-playing.
- **`similarity_search`** — manual and library implementations of L2 distance, dot product, and cosine similarity over sentence embeddings, with step-by-step comparisons using numpy, scipy, and PyTorch.
- **`data_wrangling`** — end-to-end data cleaning pipeline on a used car dataset: handling missing values (mean imputation, frequency replacement, row dropping), correcting data types, standardising units (mpg → L/100km), min-max normalisation, equal-width binning of horsepower, and one-hot encoding of categorical features with `pd.get_dummies`.
- **`langgraph_101`** — introduction to LangGraph's core primitives: typed state dictionaries, nodes, edges, routers, and conditional branching. Covers two worked examples — a user authentication workflow with success/failure routing, and a QA workflow that validates input, retrieves context, and calls an IBM Watsonx LLM to generate an answer.

**Tools:** **pandas**, **NumPy**, **matplotlib**, **LangGraph**, **LangChain**, **IBM Watsonx**
